# 4주차 실습 정리
## 오늘 한 것
- 원형 / AABB / OBB Bounding Box 시각화
- 세 방식의 충돌 판정 차이 관찰

## 관찰 내용

### 원형 vs AABB
- AABB는 히트박스 자체가 크다보니 모서리 부분으로 접근하면 가장 먼저 히트되는 모습을 보였다. 원형은 모서리 자체가 없다보니 AABB보다
  가까이에서 히트되는 모습을 볼 수 있었다.

### AABB vs OBB
- AABB는 회전하지 않고, OBB는 사각형 자체가 자연스러운 움직임으로 회전하는 모습을 볼 수 있었다. 
  
## AI와의 대화에서 배운 것
- 코드를 정리하다가 파란 원형이 사라진걸 발견하고 수정을 요청했을 때, Bounding Box 코드가 사라진 이유를 설명들었을 때 흥미로웠습니다.
- (사라진 이유: SAT 중심 구조로 바뀌면서 원형 관련 렌더링 기능은 그대로 있으나 그리기만 빠져버림 -> 수정 완료 )
  
## 내 게임에 적용한다면(어떤 방식을 쓸 것인지)
- 만들고자 하는 게임이나 상황에 따라 다르겠지만, 원형과 AABB를 많이 사용할 것 같다.
  빠르고 충분히 정확하며 유지보수가 쉬운 이 둘을 현실적으로 자주 사용하지 않을까 싶다.
